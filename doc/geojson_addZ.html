geojson_addZR Documentation

Add a Z value to geometries

Description

Use a geoserver API to recover the Z for a given set of points stored in a GeoJSON file

Usage

geojson_addZ(
  geojson.path = paste0(system.file(package = "eamenaR"),
    "/extdata/caravanserail.geojson"),
  geojson.out = NA,
  elevation.api = "open-elevation",
  sleep = 0.3,
  dirOut = paste0(system.file(package = "eamenaR"), "/results/"),
  verbose = T
)

Arguments

geojson.path

the path of the GeoJSON file. By default 'caravanserail.geojson'.

geojson.out

the name of the output file. By default NA, will add the suffix 'Z' to the the name of the GeoJSON file (variable 'geojson.path')

sleep

the time delay between two API request in seconds. By default 0.3.

dirOut

the folder where the outputs will be saved. By default: '/results'. If it doesn't exist, it will be created. Only useful is export plot is TRUE.

verbose

if TRUE (by default): verbose

map.name

the name of the output map and the name of the saved file (if export.plot is TRUE). By default "map".

ids

the IDs of the resources, by default "EAMENA.ID" (R fieldname format, without spaces).

field.names

a vector one or many field names for thematic cartography. If NA (by default), will create a general map

highlights.ids

EAMENA IDs (ex: 'EAMENA-0205783') that will be highlighted in the map. If NA (by default), no highlights.

symbology

the path to the XLSX recording the symbology for the different values, by default 'symbology.xlsx'.

stamen.zoom

the zoom of the Stamen basemap, between 0 (world, unprecise) to 21 (building, very precise). By default NA, the zoom level will be calculated automatically.

plotly.plot

if FALSE create a static PNG, if TRUE create a plotly plot as a HTML widget.

export.plot

if TRUE, export the plot, if FALSE will only display it.
